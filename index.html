How Vortex CX works

Vortex CX connects all your client communication channels into a single workspace, then uses AI to automatically extract and structure the data hidden in every conversation.

Unified Inbox

WhatsApp, SMS, email, and voice calls in a single threaded view. Agents see the full history of every client relationship across every channel.

Workflow Automation

Event-driven playbooks that route, escalate, and follow up automatically based on conversation events and client status changes.

Built-in CRM

Vortex CRM is a full CRM module built into the platform — contacts, companies, opportunities, pipeline management, and activity tracking. Data captured by AI from conversations flows directly into CRM records. Already using an external CRM? Connect Zoho, Salesforce, or others via bidirectional sync.

Email Integration

Connect Gmail or Microsoft email accounts via OAuth 2.0. Vortex CX reads incoming email from connected accounts and sends email on the user's behalf — all within the unified inbox alongside WhatsApp, SMS, and voice. Email content is processed through the same AI extraction pipeline as other channels.

Address Lookup

Google Maps Places API powers address autocomplete for contact records. Agents type a partial address and select from validated suggestions — no manual entry errors, standardized formatting, geocoded coordinates for territory management.

Mobile Ready

Native Android app with SMS and voice bridging. Your phone becomes a managed business communication hub with full conversation history.

Calendar & Booking

Schedule meetings and tasks directly from conversations. Calendar events sync with connected Google or Microsoft accounts.

AI-powered data extraction

Every client conversation contains valuable information — contact details, preferences, requirements, commitments. Traditionally, agents must manually enter this data into the CRM after every call or email. Vortex CX automates this entirely.

After each call, Vortex CX transcribes the audio using Deepgram's speech-to-text engine, then processes the transcript through an AI extraction pipeline that identifies and structures data points automatically: names, phone numbers, email addresses, product interests, compliance information, and more.

The same extraction pipeline processes email content, messaging threads, and voice recordings across all connected channels. When a client sends a new phone number in a WhatsApp message, confirms a meeting time in a Gmail thread, attaches a document to an email, or provides compliance details during a call, the platform captures and structures that data without any manual entry. Email content from connected Gmail and Microsoft accounts is read, threaded into the client's unified conversation history, and processed for data extraction alongside all other channels.

Extracted data is presented to the agent for review before it's committed to the CRM. The agent confirms, adjusts, or dismisses each extracted data point — the AI proposes, the human decides. No data is written to client records without explicit agent approval.

What the AI extracts

  • ✓ Client preferences and requirements
  • ✓ Product and service interests
  • ✓ Compliance and identity data
  • ✓ Commitments and next steps
  • ✓ Relationships between contacts
  • ✓ Sentiment and disposition

  • WhatsApp conversations
  • SMS messages

Data security and privacy by design

Vortex CX handles sensitive client data across every channel. The platform is built with security and privacy as foundational architecture decisions, not afterthoughts.

PII Vault

Personally identifiable information — phone numbers, email addresses, physical addresses — is never stored directly on operational tables. All PII is stored in an encrypted vault with AES-256 encryption at rest. Operational tables reference vault IDs only, so a database breach exposes no raw personal data.

Tenant Isolation

Every database query is filtered through Row-Level Security (RLS) policies enforced at the PostgreSQL level. Each user's JWT token carries their organization ID, and the database automatically restricts all reads and writes to that tenant's data. No application-level filtering — isolation is enforced by the database engine itself.

Encryption Everywhere

All data in transit is encrypted with TLS 1.2+. Sensitive data at rest uses AES-256 encryption. OAuth tokens, API credentials, and third-party secrets are encrypted before storage. Passwords are hashed with bcrypt. No credentials are ever stored in plaintext.

No PII in Logs

Application logs, error tracking, and audit trails never contain raw personal data. All logging uses internal subject IDs and vault references instead of phone numbers, email addresses, or names. Even if log infrastructure were compromised, no personal data would be exposed.

OAuth & Third-Party Access

When users connect Gmail or other third-party accounts, Vortex CX requests only the minimum required OAuth scopes. Tokens are encrypted at rest and used exclusively for the authorized purpose (sending/receiving email within the platform). Users can disconnect accounts at any time, and tokens are immediately revoked and deleted.

Compliance

Vortex CX is designed for GDPR, CCPA, and LGPD compliance. Data Processing Addendums are available on request. Data retention is configurable per account. Full data export and deletion capabilities ensure compliance with data subject rights requests. Audit trails track all data access and modifications.
